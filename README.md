# Лабораторная 4 (доп) — HTTP прокси с чёрным списком

Этот вариант расширяет лабу 4: добавляет **фильтрацию по чёрному списку** доменов и/или URL.

## Конфиг

Файл `config.json`:

- `blocked_domains`: домены для блокировки (точно или по суффиксу).  
  Пример: `example.com` блокирует и `example.com`, и `a.example.com`.
- `blocked_urls`: URL для блокировки по префиксу.  
  Пример: `http://live.legendy.by:8000/legendyfm` заблокирует этот ресурс (и всё, что начинается с этого префикса).

## Запуск

```bash
python3 http_proxy_blacklist.py --listen-host 127.0.0.1 --listen-port 8080 --config config.json
```

## Поведение при блокировке

Если запрошенный ресурс попадает под правила, прокси вернёт:

- `HTTP/1.1 403 Forbidden`
- HTML-страницу с текстом “Доступ к ресурсу заблокирован” и URL.

Лог в консоли будет содержать отметку:

`=> 403 (BLOCKED)`
